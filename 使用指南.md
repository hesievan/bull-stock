## 快速开始

### 前置条件

- Python 3.10+
- tushare 账号（2000+ 积分）
- SQLite（系统自带）

### 安装

```bash
git clone <repo_url>
cd bull-market-heat-index
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env，填入你的 TUSHARE_TOKEN
```

### 验证

```bash
python -m pytest tests/ -v
python scripts/run_daily.py
```

## 每日运行

### 运行命令

```bash
# 计算今日
python scripts/run_daily.py

# 计算指定日期
python scripts/run_daily.py 2026-06-18
```

### 输出示例

```
2026-06-24 16:38:46,883 [INFO] RUN SUMMARY: 20 OK / 0 FAILED / 3 SKIPPED (22.6s)
```

所有 Step 独立 try/except，单个 Step 失败不阻断后续流程。

## 指标体系

### V2 引擎（9 指标 / 4 维度 — 每日流水线所用）

| 维度 | 权重 | 子指标 | 说明 | 方向 |
|------|------|-------|------|------|
| **估值** | 40% | 大盘PE | 全市场PE中位数历史百分位 | 越高越贵 |
| | | 股权风险溢价ERP | 1/PE - 10Y国债收益率 | 越低越贵 |
| | | 巴菲特指标 | A股总市值 ÷ 年度GDP | 越高越贵 |
| **资金** | 30% | 两融余额市值比 | (融资+融券)÷流通市值 | 越高越热(>90%分位转减) |
| | | 存款市值比 | M2 ÷ 总市值 | 越低=资金搬家越热 |
| **情绪** | 20% | 成交额M2比 | 日成交额 ÷ M2 | 越高越热 |
| | | 换手率 | 成交额÷流通市值×10 | 越高越热 |
| **结构** | 10% | 创新高占比 | 250日新高股票占比 | 越高越好 |
| | | MA排列比 | MA20>MA60>MA120占比 | 越高越好 |

**展示不计分**: QVIX恐慌指数

### 评分合成路径

```
原始数据 → 10年历史百分位(0-100) → 指标加权合成综合分 → 3日平滑 → 四区间分类
```

### 背离惩罚

- **情绪背离**: 高换手率 + 指数下跌 → 情绪得分扣减最多20分
- **新高顶背离**: 指数涨 + 新高占比下降且<30% → 结构分扣减最多15分

### 热度阈值

| 颜色 | 分数 | 含义 |
|------|------|------|
| 🟢 绿色 | 0–40 | 估值合理，情绪冷淡 |
| 🟡 黄色 | 40–55 | 部分指标偏高 |
| 🟠 橙色 | 55–65 | 多项指标偏高，考虑减仓 |
| 🔴 红色 | 65–100 | 多项指标历史高位，考虑离场 |

## 指数牛市见顶预判

### 目标指数

| 指数 | tushare代码 | akshare代码 | 数据起始 |
|------|------------|------------|---------|
| 沪深300 | 000300.SH | sh000300 | 2014-12-29 |
| 创业板指 | 399006.SZ | sz399006 | 2014-12-29 |
| 科创50 | 000688.SH | sh000688 | 2020-01-02 |
| 北证50 | 899050.BJ | bj899050 | 2022-12-19 |
| 中证A500 | 000510.SH | sh000510 | 2024-09-23 |
| 中证1000 | 000852.SH | sh000852 | 2014-12-29 |

### 评分体系

每个指数独立计算:

**技术指标（4个子指标）**: MA250偏离度、20日动量、60日动量、120日动量

**估值指标（2个子指标）**: PE TTM 分位、PB 分位（仅沪深300、创业板指可用）

```
composite = tech_avg × 0.5 + val_avg × 0.5  # 有估值数据时
composite = tech_avg                           # 无估值数据时
```

### 信号阈值

与全市场热度一致：≥65 🔴 红 | ≥55 🟠 橙 | ≥40 🟡 黄 | <40 🟢 绿

### 使用建议

- **全市场热度为宏观择时参考**，指数热度用于判断具体板块风险
- 当全市场热度 ≥ 65（红区）且多个指数同时红区 → 系统性风险较高
- 创业板和科创50同时红区 → 成长股泡沫风险

## 结果文件说明

所有计算结果输出到 `web/data/` 目录：

| 文件 | 说明 |
|------|------|
| `index.json` | 最新热度（综合 + 4维度 + 9指标原始值） |
| `detail.json` | 含完整指标明细 |
| `history.json` | 历史热度序列 |
| `indicator_history.json` | 9指标历史趋势（供前端图表） |
| `sectors.json` | 板块热度排名（CSRC行业分类） |
| `run_status.json` | 各Step执行状态 |

### index.json 格式

```json
  "composite_score": 54.0,
  "composite_score_smoothed": 52.8,
  "level": "yellow",
  "level_smoothed": "yellow",
  "dimensions": {
    "valuation": { "score": 53.8, "label": "估值" },
    "fund":       { "score": 56.1, "label": "资金" },
    "sentiment":  { "score": 79.4, "label": "情绪" },
    "structure":  { "score": 14.2, "label": "结构" }
  },
  "indicators_v2": {
    "pe": 23.88,
    "erp": 2.44,
    "buffett": 0.95,
```

## 常见问题

### Q: 运行报 `TUSHARE_TOKEN` 未设置

A: 确保 `.env` 文件存在且内容正确：
```
TUSHARE_TOKEN=你的token
```
系统还自动搜索 `~/daily_stock_analysis/.env` 作为备选。

### Q: 某个 Step 一直 FAILED

A: 检查网络连接和 tushare 积分余额：
```bash
cat web/data/run_status.json
```
常见原因：
- tushare 积分不足（需要 2000+）
- 网络超时（重试即可）
- 非交易日运行

### Q: 为什么某些指数没有 PE/PB 评分

A: tushare 的 `index_dailybasic` 仅覆盖沪深300和创业板指等少数指数。科创50/北证50/A500/中证1000 没有指数级 PE/PB 数据，使用纯技术评分。

### Q: 非交易日能运行吗

A: 可以，但部分需要日频数据的步骤会 SKIP。S5_calc 仍可计算（使用最新可用数据）。

### Q: 数据库太大怎么办

A: 运行 `python scripts/db_maintenance.py --vacuum` 压缩。
