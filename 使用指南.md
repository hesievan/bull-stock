## 快速开始

### 前置条件

- Python 3.10+
- tushare 账号（2000+ 积分）
- SQLite（系统自带）

### 安装

```bash
# 克隆仓库
git clone <repo_url>
cd bull-market-heat-index

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置 tushare token
cp .env.example .env
# 编辑 .env，填入你的 TUSHARE_TOKEN
```

### 验证

```bash
# 运行测试
python -m pytest tests/ -v

# 计算今天的热度
python scripts/run_daily.py
```

## 每日运行

### 运行命令

```bash
# 计算今日
python scripts/run_daily.py

# 计算指定日期
python scripts/run_daily.py 2026-06-18

# 指定数据库路径
python scripts/run_daily.py --db /path/to/heat_index.db
```

### 输出示例

```
2026-06-20 22:59:39,894 [INFO] RUN SUMMARY: 17 OK / 0 FAILED / 3 SKIPPED (41.1s)
```

所有 Step 独立 try/except，单个 Step 失败不阻断后续流程。

### 流水线步骤

| Step | 名称 | 说明 | 耗时参考 |
|------|------|------|---------|
| S0 | init_db | 数据库建表/迁移 | <0.1s |

| S26 | S26_circ_mv | 全市场流通市值（供融资余额比） | 0.5–2s |

| **S24** | **precompute_check** | **预计算表陈旧检测 (v3.11)** | **<0.1s** |
| S3 | tushare | 融资融券 + 北向 + 国债 | 1–3s |
| S4 | ah_premium | AH 溢价指数 | 1–2s |
| **S5** | **S5_calc** | **19 子指标 + 新鲜度权重衰减** | **5–15s** |
| **S55** | **S55_index_heat** | **六大指数牛市见顶预判 (含相对强弱)** | **<0.1s** |

| S8 | final_save | 最终保存 | <0.1s |
| S9 | notify | 飞书推送（含新鲜度告警） | <0.5s |

## 指标体系

### 维度权重

| 维度 | 权重 | 子指标数 | 核心数据 |
|------|------|---------|---------|
| 估值 | 25% | 3 | PE/PB分位、破净率、ERP |
| 宏观 | 15% | 2 | M1-M2剪刀差、M2同比 |
| 资金 | 15% | 2 | 北向变化率、融资余额比 |
| 情绪 | 20% | 5 | 换手率、涨跌家数比、涨停占比、涨跌停比、QVIX |
| 技术 | 10% | 5 | MA排列比、均线偏离度、20/60/120日动量 |
| 结构 | 15% | 2 | 创新高比例、AH溢价 |

### 评分合成路径

```
原始数据 → 历史10年分位(0-100) → 维度内等权平均 → 加权合成综合分
```

### 热度阈值

| 颜色 | 分数 | 含义 |
|------|------|------|
| 🟢 绿色 | 0–40 | 估值合理，情绪冷淡 |
| 🟡 黄色 | 40–55 | 部分指标偏高 |
| 🟠 橙色 | 55–65 | 多项指标偏高，考虑减仓 |
| 🔴 红色 | 65–100 | 多项指标历史高位，考虑离场 |

### 每个子指标的计算方法

详见 `src/indicators/` 下各模块文件。

## 指数牛市见顶预判

### 目标指数

| 指数 | 代码(tushare) | 代码(akshare) | 数据起始 |
|------|--------------|--------------|---------|
| 沪深300 | 000300.SH | sh000300 | 2014-12-29 |
| 创业板指 | 399006.SZ | sz399006 | 2014-12-29 |
| 科创50 | 000688.SH | sh000688 | 2020-01-02 |
| 北证50 | 899050.BJ | bj899050 | 2022-12-19 |
| 中证A500 | 000510.SH | sh000510 | 2024-09-23 |
| 中证1000 | 000852.SH | sh000852 | 2014-12-29 |

### 评分体系

每个指数独立计算 4 类技术指标 + 2 类估值指标：

**技术指标（4 个子指标）**

| 指标 | 计算方式 | 条件 |
|------|---------|------|
| MA250 偏离度 | (收盘价/250日均线-1) 历史分位 | 需 ≥200 个有效偏离值 |
| 20日动量 | 20日涨跌幅历史分位 | 需 ≥40 个交易日数据 |
| 60日动量 | 60日涨跌幅历史分位 | 需 ≥120 个交易日数据 |
| 120日动量 | 120日涨跌幅历史分位 | 需 ≥240 个交易日数据 |

**估值指标（2 个子指标）**

| 指标 | 计算方式 | 可用指数 |
|------|---------|---------|
| PE TTM 分位 | PE 历史分位 | 沪深300、创业板指 |
| PB 分位 | PB 历史分位 | 沪深300、创业板指 |

> 沪深300 和 创业板指 有 tushare `index_dailybasic` 提供的 PE/PB 数据；
> 科创50/北证50/A500/中证1000 无指数级 PE/PB 数据源，估值评分为 null。

**复合评分公式**

```
tech_avg = 技术指标等权平均
val_avg  = 估值指标等权平均 (若无可则为 null)

composite = tech_avg * 0.5 + val_avg * 0.5   # 有估值数据时
composite = tech_avg                           # 无估值数据时
```

### 信号阈值

与全市场热度一致：≥65 🔴 红 | ≥55 🟠 橙 | ≥40 🟡 黄 | <40 🟢 绿

### 量能评分说明

volume 评分（近 60 日成交额均值比）仅作为参考信息留在 `detail` 字段中，**不参与技术综合评分**。原因是 tushare 提供的 `index_daily` 量价数据在 2025 年中之后的单位存在异常（日成交额从正常万亿级别降到百亿级别），若纳入综合会拉低指数评分。

### 读取结果

```bash
cat web/data/index_heat.json
```

输出为 JSON 数组，每项包含：

| 字段 | 说明 |
|------|------|
| `index_code` | akshare 指数代码 |
| `index_name` | 中文名称 |
| `trade_date` | 交易日 |
| `close` | 收盘价 |
| `pct_change` | 当日涨跌幅(%) |
| `tech_score` | 技术综合分 (0–100) |
| `val_score` | 估值综合分 (0–100) |
| `composite_score` | 复合得分 (0–100) |
| `level` | 信号级别 (green/yellow/orange/red) |
| `n_tech_indicators` | 使用的技术指标数量 |
| `n_val_indicators` | 使用的估值指标数量 |
| `detail` | 各子指标明细 |

### 使用建议

- **全市场热度为宏观择时参考**，指数热度用于判断具体板块风险
- 当全市场热度 ≥ 65（红区）且多个指数同时红区 → 系统性风险较高
- 创业板和科创50同时红区 → 成长股泡沫风险
- 北证50独绿 → 小盘价值尚未过热

## v3.11 新功能

### 市场结构断点检测

替代固定10年评分窗口，使用 **CUSUM 变点检测** 自动识别均值移位：

- 当检测到市场结构变化（如注册制改革、量化新规），自动截断远古数据
- 仅在"当前市场结构"有效期内计算百分位
- 消除2015年牛市等极端值对当前评分的稀释
- 文件: `src/indicators/regime_detector.py`

### 数据新鲜度与权重衰减

月频数据插值到日频会扭曲当日评分。系统检测每个维度指标的实际数据日期：

- 各指标声明最大允许延迟（macro 22天，其他 3-5 天）
- 超出后按指数衰减（decay_rate=0.15）降低该维度权重
- 衰减的权重按新鲜度比例重新分配给其他维度
- 输出 `effective_weights` 到 index.json
- 文件: `src/data/freshness.py`

### 短历史指数相对强弱评分

北证50（~850行）和中证A500（~420行）数据量不足以支持120日动量分位计算。

改用 **相对强弱评分**：
- 计算目标指数 vs 沪深300 的 60日滚动超额收益
- 在超额收益的自身短历史内评分
- 连续跑赢天数额外加分（最多+15）
- 信号从"随机噪声"变为"有统计意义的相对强弱判断"

### 数据质量报告

每日输出 `data_quality` 区块到 index.json 和飞书通知：

```json
{
  "overall_quality": "good",
  "dimensions": {
    "macro": { "available": 2, "total": 2, "freshness": 0.5, "status": "degraded" }
  },
  "missing_indicators": [],
  "stale_dimensions": ["macro"]
}
```

- `overall_quality`: good / degraded / poor
- 每维度: 子指标可用率 + 新鲜度
- 飞书通知中高亮显示 stale 维度

### 预计算表陈旧检测

新增 S24_precompute_check 步骤，在 S5 计算前检测10张预计算表的陈旧状态：

- 每张表配置允许最大陈旧天数 + 是否有 fallback
- Stale 表打 warning 日志
- 完整结果写入 run_status.json

### 极端值自动检测

`utils._clip_outliers()`: 超出 mean ± 5σ 的值自动截断到边界值。


- 综合热度 + 信号等级 + 颜色图标

## 常见问题

### Q: 运行报 `TUSHARE_TOKEN` 未设置

A: 确保 `.env` 文件存在且内容正确：
```
TUSHARE_TOKEN=你的token
```

系统还自动搜索 `~/daily_stock_analysis/.env` 作为备选。

### Q: 某个 Step 一直 FAILED

A: 检查网络连接和 tushare 积分余额：
```bash
# 查看今日运行状态
cat web/data/run_status.json
```

常见原因：
- tushare 积分不足（需要 2000+）
- 网络超时（重试即可）
- 非交易日运行（S4 AH溢价等步骤会 SKIP）

### Q: 科创50/北证50 在 index_heat.json 中显示 "unknown"

A: 这些指数的 `index_daily` 数据需要至少首日拉取。运行一次 `run_daily.py` 后即可自动补全。

### Q: 为什么某些指数没有 PE/PB 评分

A: tushare 的 `index_dailybasic` 仅覆盖沪深300（000300.SH）和创业板指（399006.SZ）等少数指数。科创50/北证50/A500/中证1000 没有指数级 PE/PB 数据，使用纯技术评分。

### Q: index_heat 的 volume 评分为什么是 0.5

A: 见 [量能评分说明](#量能评分说明)。tushare `index_daily` 的量价数据在近半年单位异常，该评分仅供参考，不参与综合。

### Q: 如何自定义热度阈值

A: 修改 `src/output/json_writer.py` 中的 `get_heat_level()` 函数，或 `src/indicators/index_heat.py` 中的 `get_overheat_level()` 函数。

### Q: 非交易日能运行吗

A: 可以，但 S4（AH溢价）等需要日频数据的步骤会 SKIP。S5_calc 仍可计算（使用最新可用数据）。

### Q: index.json 中的 data_quality 是什么意思

A: 数据质量报告，包含：
- `overall_quality`: good（全部正常）/ degraded（部分降级）/ poor（关键数据缺失）
- `dimensions[].freshness`: 0–1，越接近1越新鲜
- `dimensions[].status`: ok / degraded / poor
- 飞书通知中会高亮显示 degraded 或 poor 的维度

### Q: 数据库太大怎么办

A: 运行 `python scripts/db_maintenance.py --vacuum` 压缩。如需归档旧数据：`python scripts/db_maintenance.py --archive 2020`。
